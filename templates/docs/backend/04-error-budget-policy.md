# Error Budget Policy

## Scope

- Link each governed service, user journey, SLI, SLO, and source decision.

## Budget Actions

- Define owned actions for healthy, at-risk, and exhausted error budgets.

## Release Policy

- Define how burn-rate evidence changes release, rollback, and feature-flag decisions.

## Incident Policy

- Define paging, ticketing, escalation, and incident follow-up expectations.

## Review

- Define the review cadence, participants, evidence window, and target-change process.
